هذا المستند يشرح بالتفصيل المنطق البرمجي (Logic) والقواعد اللغوية (Syntax) للأجزاء التي قام عمر بتنفيذها في مشروع نظام إدارة المستشفى.

## 1. كلاس المدير (Admin Class)
قام عمر ببرمجة كلاس `Admin` المسؤول عن إدارة الأطباء والمرضى.

### أ- ملف `Admin.h` (التصميم)
*   **الوراثة (Inheritance):**
    ```cpp
    class Admin : public User
    ```
    *   **المنطق:** المدير هو نوع من أنواع المستخدمين، لذا يرث الصفات الأساسية (مثل الاسم وكلمة السر) من كلاس `User`.
*   **العمليات (Methods):** تم تعريف دوال لإضافة وحذف الأطباء والمرضى.

### ب- ملف `Admin.cpp` (التنفيذ)
*   **إضافة طبيب/مريض (`addDoctor` / `addPatient`):**
    *   **المنطق:** نتحقق أولاً إذا كان هناك مكان في المصفوفة (أقل من 100). إذا توفر مكان، نقوم بإنشاء كائن جديد باستخدام `new` ونخزنه في المصفوفة.
    *   **السينتاكس:** 
        ```cpp
        doctors[count++] = new Doctor(...);
        ```
*   **حذف طبيب/مريض (`removeDoctor` / `removePatient`):**
    *   **المنطق:** نبحث عن الـ ID المطلوب. عند إيجاده، نقوم بحذف الكائن من الذاكرة (`delete`) لتوفير المساحة، ثم نقوم بتزييح (Shift) العناصر المتبقية لسد الفجوة في المصفوفة.
    *   **السينتاكس:** 
        ```cpp
        delete doctors[i]; // مسح من الذاكرة
        for (int j = i; j < count - 1; j++) doctors[j] = doctors[j + 1]; // تزييح العناصر
        ```

## 2. نظام حفظ واسترجاع البيانات (File I/O) - [الميزة الإضافية]
قام عمر بتنفيذ نظام لضمان عدم ضياع البيانات عند إغلاق البرنامج باستخدام مكتبة `<fstream>`.

### أ- دالة الحفظ (`saveData`)
*   **المنطق:** تمر الدالة على مصفوفة الأطباء ومصفوفة المرضى، وتكتب بيانات كل شخص في ملف نصي (`.txt`).
*   **السينتاكس:**
    ```cpp
    ofstream docFile("doctors.txt"); // فتح ملف للكتابة
    docFile << doctors[i]->getId() << " " << ... ; // كتابة البيانات مفصولة بمسافات
    ```

### ب- دالة التحميل (`loadData`)
*   **المنطق:** عند تشغيل البرنامج، تقرأ هذه الدالة البيانات من الملفات النصية وتُعيد إنشاء الأجسام (Objects) في الذاكرة.
*   **السينتاكس:**
    ```cpp
    ifstream docFile("doctors.txt"); // فتح ملف للقراءة
    while (docFile >> id >> user >> pass >> ...) // القراءة حتى نهاية الملف
    ```

## 3. حلقة التشغيل الأساسية (Main Application Loop)
قام عمر بربط جميع أجزاء المشروع معاً في ملف `HospitalManager.cpp`.

*   **المنطق:** 
    1.  تحميل البيانات المحفوظة في البداية (`loadData`).
    2.  عرض القائمة الرئيسية للمستخدم (مدير، طبيب، مريض).
    3.  التحقق من بيانات الدخول (Login).
    4.  توجيه المستخدم للقائمة الخاصة به.
    5.  حفظ البيانات عند الخروج (`saveData`).
*   **السينتاكس:**
    *   استخدام `do-while loop` لضمان استمرار البرنامج حتى يختار المستخدم "خروج" (Exit).
    *   تنظيف الذاكرة في النهاية باستخدام `delete` لكل المؤشرات (Pointers) لتجنب حدوث `Memory Leak`.
